## ğŸ” What You'll See in Firestore:

### Collection: "users"
This is like a folder containing all students

### Documents: Individual Students
Each student has their own document (like a file)
- Document ID = Student's name
- Contains their test data

### Fields in Each Document:
- **name**: Student's name (string)
- **tests**: Array of all their test attempts

### Each Test Contains:
- **operation**: addition, subtraction, multiplication, or division
- **difficulty**: 1digit, 2digit, or 3digit

- **percentage**: Score percentage (0-100)

## ğŸ” Firebase Security Rules:

Currently set to **test mode** (open access). This is fine for now, but for production you should secure it.

### Current Rules (Test Mode):
```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /{document=**} {
      allow read, write: if request.time < timestamp.date(2025, 11, 10);
    }
  }
}
```

This allows anyone to read/write until the expiration date.

### Recommended Production Rules:
```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /users/{userId} {
      allow read: if true;  // Anyone can read
      allow write: if true; // Anyone can write (for student accounts)
    }
  }
}
```

To update rules:
1. Go to Firestore Database
2. Click "Rules" tab at top
3. Edit the rules
4. Click "Publish"
